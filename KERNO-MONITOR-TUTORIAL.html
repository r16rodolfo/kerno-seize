Kerno Monitor

De ferramentas prontas a um painel com IA consultora. Evolua conforme sua agência cresce.

01 Por que NÃO construir o painel agora

Antes de investir dias construindo, entenda o que já existe de graça:

FerramentaPlano FreePara 4-6 apps
Sentry5K erros/mês, projetos ilimitados, 1 usuário✅ Uma conta resolve tudo
UptimeRobot50 monitores, check 5min✅ 3 monitores/app = 12-18 usados de 50
GitHubActions + Dependabot + Secrets ilimitados✅ Sem limites
Cloudflare1 domínio/conta, CDN+WAF+DDoS✅ 1 conta free por domínio
💰 Custo total com 4-6 projetos: R$ 0. Não precisa criar conta por projeto. Uma conta de cada basta.

Construir um painel customizado agora seria gastar dias replicando dashboards que o Sentry e UptimeRobot já oferecem. O que faz sentido é usar as ferramentas prontas AGORA e construir o painel QUANDO os diferenciais reais justificarem.

Os 3 diferenciais reais (que nenhuma ferramenta pronta oferece)

1. Audit logs centralizados — Nenhuma ferramenta monitora as tabelas audit_log do Supabase de múltiplos projetos.

2. Claude como consultor de segurança — Um painel que não só mostra dados mas INTERPRETA e sugere ações usando IA.

3. Verificação automatizada de RLS — Nenhuma ferramenta verifica periodicamente se suas RLS policies estão corretas.

02 Roadmap em 3 Fases

🟢 Fase 1 — Ferramentas Prontas

AGORA · 0-3 projetos

Use as ferramentas gratuitas com uma conta cada. "Painel" = 3 abas fixas no navegador.

📊
Sentry: Uma conta, um projeto por app. Dashboard nativo mostra todos juntos. Alertas por email.
⏱️
UptimeRobot: Uma conta, 3 monitores por app. Dashboard nativo + app mobile. Alertas por email e push.
🔍
GitHub: Notifications centralizadas (Actions + Dependabot + Secrets) para todos os repos.
📝
Audit Logs: Consulta manual semanal no Supabase SQL Editor de cada projeto.
📋
Incidentes: Siga os playbooks do Guia de Resposta a Incidentes.
✅ Custo: R$ 0 · Tempo de setup: ~1h total · Resultado: monitoramento completo para até 6 projetos.

🔵 Fase 2 — Kerno Monitor (MVP com IA)

QUANDO TIVER · 3+ projetos ativos

Construir no Lovable um painel que foca nos diferenciais reais: audit logs centralizados + Claude como consultor.

🤖 Feature central: Claude como consultor de segurança

O painel coleta dados de todos os projetos e envia para a API do Claude, que analisa e responde perguntas como:

"O App Financeiro teve 23 deleções na tabela transactions vindas do usuário X nas últimas 6 horas. O padrão normal é 2-3 deleções por dia. Recomendo: (1) verificar se o usuário X tem permissão para deletar transactions, (2) revisar a RLS policy da tabela, (3) verificar no audit_log os old_data para saber o que foi deletado."

📡
Audit logs centralizados: Edge Functions em cada projeto enviam resumos para o Supabase do Monitor.
🤖
Claude AI: Analisa dados, detecta anomalias, sugere ações. Usa API do Claude (Anthropic).
📊
Dashboard unificado: Cards de status, feed de eventos, gráficos. Dark mode. Auto-refresh.
🔍
RLS Checker: Verificação periódica automatizada de RLS em todas as tabelas de todos os projetos.
💡 A Fase 2 usa dados do UptimeRobot e Sentry via APIs (que continuam no free tier). O Monitor COMPLEMENTA, não substitui.

🟣 Fase 3 — Monitor Avançado

FUTURO · 5+ projetos · equipe

Automações avançadas e testes periódicos.

🧪
DAST automatizado: Testes de acesso não autorizado e XSS rodando periodicamente.
📧
Alertas inteligentes: Claude analisa em tempo real e envia alertas com contexto e sugestão de ação.
📄
Relatórios automáticos: Relatório semanal gerado por IA com status de todos os projetos.
🏷️
Status pages por cliente: Página pública mostrando uptime de cada projeto (para o cliente ver).
03 PRD da Fase 2 — Kerno Monitor com Claude AI

Quando tiver 3+ projetos ativos e a Fase 1 funcionando, crie um projeto Lovable SEPARADO com este PRD:

## PRD — Kerno Monitor v1: Painel com IA Consultora

### Visão
Painel centralizado de segurança para agência que gerencia
múltiplas aplicações Lovable+Supabase. O diferencial é a
integração com Claude AI que analisa dados e sugere ações.

### Stack
React + TypeScript + Tailwind + shadcn/ui + Recharts
Backend: Supabase separado (novo projeto)
IA: API do Claude (Anthropic) via Edge Function
Deploy: Vercel

### Autenticação
Login email/senha. Sem registro público. Roles: admin, viewer.

### Banco de Dados

Tabela `projects`:
- id, name, url, supabase_url, supabase_service_role_encrypted
- github_repo, sentry_project_slug, uptimerobot_monitor_id
- status (online/degraded/offline/unknown)
- last_rls_check, last_rls_result (json)
- created_at, updated_at

Tabela `audit_summaries` (dados vindos de cada projeto):
- id, project_id (FK), period_start, period_end
- total_actions, inserts, updates, deletes
- top_users (jsonb), top_tables (jsonb)
- anomalies_detected (jsonb)
- fetched_at

Tabela `ai_analyses`:
- id, project_id (FK), timestamp
- context (jsonb — dados enviados ao Claude)
- analysis (text — resposta do Claude)
- severity (info/warning/critical)
- recommended_actions (jsonb)
- acknowledged (boolean)

Tabela `rls_checks`:
- id, project_id (FK), checked_at
- tables_checked (integer)
- tables_with_rls (integer)
- tables_without_rls (jsonb — lista das tabelas expostas)
- policies_summary (jsonb)
- status (ok/warning/critical)

Tabela `events`:
- id, project_id (FK), timestamp
- severity, source, title, description
- resolved, resolved_at

Tabela `api_keys`:
- id, service, encrypted_key (Supabase Vault)

### Páginas

/dashboard — Cards de status de cada projeto com semáforo
(verde/amarelo/vermelho), contadores globais, feed de eventos,
e caixa de IA onde posso perguntar sobre qualquer projeto.
Auto-refresh 60 segundos.

/projects/:id — Detalhe: uptime 7d, erros Sentry 7d,
último resultado do RLS checker, resumo do audit_log,
últimas análises da IA, e botão "Pedir análise ao Claude".

/ai — Chat com Claude sobre segurança. Ele tem acesso a
todos os dados dos projetos. Exemplos de perguntas:
"Algum projeto teve atividade anômala essa semana?"
"Quais tabelas do App Financeiro não têm RLS?"
"Resuma os incidentes da última semana e sugira melhorias"

/settings — API keys, gerenciar projetos, gerenciar usuários.

### Edge Functions

1. `fetch-audit-summary` — Para cada projeto, conecta no
   Supabase dele e traz resumo do audit_log.
   Roda a cada 6 horas via pg_cron.

2. `check-rls` — Para cada projeto, executa query que
   lista tabelas e status do RLS. Roda diariamente.

3. `analyze-with-claude` — Pega os dados mais recentes de
   um projeto (audit summary + RLS check + últimos eventos)
   e envia para a API do Claude pedindo análise de segurança.
   Salva resposta em ai_analyses.

4. `fetch-external-apis` — Busca dados do UptimeRobot e
   Sentry via APIs públicas. Roda a cada 5 minutos.

### Integração Claude AI

System prompt para o Claude:
"Você é um consultor de segurança analisando dados de
aplicações web. Você recebe: audit logs, status de RLS,
erros do Sentry, e uptime. Sua tarefa é: (1) identificar
anomalias, (2) classificar severidade, (3) recomendar
ações concretas. Seja direto e prático. Se algo parece
normal, diga que está normal."

Modelo: claude-sonnet-4-20250514
Max tokens: 1000 por análise

### Design
Dark mode. Fundo quase preto. Cards com bordas coloridas
por status. JetBrains Mono para métricas. Otimizado para
monitor 1920x1080 (modo sala de monitoramento com F11).
Responsivo para mobile (verificação rápida).

### Segurança
RLS em todas as tabelas. API keys no Vault.
service_role dos projetos monitorados criptografadas.
Auth obrigatório. Headers no vercel.json.
⚠️ Sobre o custo da API do Claude: O modelo Sonnet é o mais custo-efetivo. Uma análise por projeto por dia = ~4 análises/dia = custo estimado de $1-3/mês. Bem menor que qualquer ferramenta paga de monitoramento.
